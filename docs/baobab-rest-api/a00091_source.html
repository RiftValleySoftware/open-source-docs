5 BASALT PLACES REST PLUGIN
6 =========================
7 
8 Part of the BASALT Extension Layer, Which is Part of the BAOBAB Server, which is part of the Rift Valley Platform
9 -----------------------------------------------------------------------------------------------------------------
10 ![BAOBAB Server and The Rift Valley Platform](images/BothLogos.png)

12 INTRODUCTION
13 ============
14 
15 The Places Plugin is a basic \ref BASALT [REST](https://restfulapi.net) Plugin; part of the "standard" set.
16 
17 You use this plugin to obtain listings/details of places (locations and addresses) in the BAOBAB server.
18 
19 USAGE
20 =====
21 
22 This plugin is accessed by setting `"places"` as the Command in the [REST](https://restfulapi.net) URI. There are a number of other aspects to the URI that will be explained:
23 
24  {GET|POST|PUT|DELETE} http[s]://{SERVER URL}/{json|xml|xsd}/places/[INTEGER RESOURCE IDS CSV][?{show_details|show_parents|SEARCH PARAMETERS -DISCUSSED BELOW|PUT/POST PARAMETERS -DISCUSSED BELOW}]
25 
26 MISCELLANEOUS PARAMETERS
27 ------------------------
28 These are additional parameters that you can use to specify various formats and information fields for resources.
29 
30 - `writeable`
31 
32  *No Value Required -Just Add the Query.* If you add this, then only resources that can be modified by the current logged-in user will be returned. This will apply to GET, PUT and DELETE.
33 
34  **NOTE:** It goes without saying (but we're saying it anyway) that this will only be useful when logged in with an API Key.
35 
36 - `show_details`
37 
38  *No Value Required -Just Add the Query.* If you add this, then resource records will be returned, showing as much information about the resources as possible (in GET method). Otherwise, you will receive an abbreviated response.
39 
40 - `show_parents`
41 
42  *No Value Required -Just Add the Query.* If you add this, then resource records will be returned, showing as much information about the resources as possible (in GET method), and they will also list any "parent" records that have the given record as a "child." It should be noted that specifying this can add considerable overhead to the call; slowing it down significantly. It's really designed for "focused" resource information.
43 
44 SEARCH PARAMETERS
45 -----------------
46 **Locality Radius Searches**
47 All three of these must be used together. If you specify them, then any place resources that have a long/lat that falls within the radius will be returned.
48 Resources without long/lat will not be returned.
49 
50 - `search_longitude=`
51 
52  *Floating-Point Decimal Value.* This is the longitude of the resource, in degrees.
53 
54 - `search_latitude=`
55 
56  *Floating-Point Decimal Value.* This is the latitude of the resource, in degrees.
57 
58 - `search_radius=`
59 
60  *Floating-Point Decimal Value.* This is a value for a radius (not diameter) circle around the given longitude and latitude. It is in Kilometers.
61  If the resource is "fuzzy," it is possible that the long/lat shown by the resource (which is deliberately inaccurate) may be outside the radius, but the actual resource location is within the radius.
62 
63 - `search_address=`
64 
65  *String.* If the server is set up for geocoding, then this can be used instead of `search_longitude` and `search_latitude` (you still need `search_radius`, though). It can contain an address that will be sent to Google for geocoding.
66  This is likely to only be available to logged-in users, although it is possible for the server to be configured to allow all users to use the facility.
67 
68 **String Searches**
69 These searches allow you to specify a simple case-insensitive string value for the indicated resource column. You can use SQL-style wildcards (%) in the strings. Not specifying a value, but specifying the parameter name and equals sign indicates that the indicated field MUST be empty. Not specifying a field at all indicates that the value of that field is not considered in the search.
70 These are actual string matches. They do not do an address lookup, and resources that don't have long/lat can be returned in these searches.
71 
72 - `search_name=`
73 
74  *String.* This is a simple resource name.
75 
76 - `search_venue=`
77 
78  *String.* This is an arbitrary string value that names the venue of any address.
79 
80 - `search_street_address=`
81 
82  *String.* This is an arbitrary string value that names the street address of any address.
83 
84 - `search_extra_information=`
85 
86  *String.* This is an arbitrary string value that is for the "extra information" section of any address.
87 
88 - `search_town=`
89 
90  *String.* This is an arbitrary string value that names the town/city/municipality of any address.
91 
92 - `search_county`
93 
94  *String.* This is an arbitrary string value that names the county or sub-province of any address.
95 
96 - `search_state=`
97 
98  *String.* This is an arbitrary string value that names the state or province of any address.
99 
100 - `search_postal_code=`
101 
102  *String.* This is an arbitrary string value that names the postal code of any address.
103 
104 - `search_nation=`
105 
106  *String.* This is an arbitrary string value that names the nation of any address.
107 
108 - `search_tag8=`
109 
110  *String.* This is an arbitrary string value that can be applied to the place.
111 
112 - `search_tag9=`
113 
114  *String.* This is an arbitrary string value that can be applied to the place.
115 
116 SPECIAL FORMATS OF RESPONSE DATA
117 --------------------------------
118 **Resource Response Format Query Parameters**
119 You can ask that data be returned in integers (individual resource IDs), as opposed to ID records, or even as a single integer, which specifies the number of resources that would be returned for the given search. In either case, the responses are wrapped in the requested data format (JSON or XML).
120 
121 **NOTE:** These can only be used for GET method.
122 
123 - `search_ids_only`
124 
125  *No Value Required -Just Add the Query.* If you add this, then the response will be arrays of integers, which will represent resource IDs (which can be used in resource specifiers in subsequent calls).
126 
127 - `search_count_only`
128 
129  *No Value Required -Just Add the Query.* If you add this, then the response will be a single integer. It will be the number of resources that would be returned by the specified search.
130 
131 PAGING OF RESPONSE DATA
132 -----------------------
133 **Paging Query Parameters**
134 These can be used for GET, PUT and DELETE.
135 
136 It is possible to request that the data be returned in "pages," where you specify a "page size," and then a "requested page of results." This allows you to focus on only a part of a large dataset, or have the response sent back in manageable-size pieces.
137 
138 Paging is another place we deviate from standard REST. In REST, you usually indicate pages in the resource request, but we specify paging via query parameters:
139 
140 These apply to both normal resource response and ID-only resource response.
141 
142 - `search_page_size=`
143 
144  *Integer.* This is the number of resources to send per page. If the page size is greater than the available resources for that page, then the number of resources returned for that page will be fewer than the specified page size.
145 
146 - `search_page_number=`
147 
148  *Integer.* This is the 0-based index of the requested page. Only the resources for that single page will be returned. If the index is greater than the number of available pages, then nothing will be returned.
149 
150 POST AND PUT PARAMETERS
151 -----------------------
152 There are parameters that can be used to set data when creating new places (POST), or editing existing ones (PUT):
153 
154 - `name=`
155 
156  *String.* This is a simple resource name.
157 
158 - `lang=`
159 
160  *String.* This is the language identifier.
161 
162 - `longitude=`
163 
164  *Floating-Point Decimal Value.* This is the longitude of the resource, in degrees.
165 
166 - `latitude=`
167 
168  *Floating-Point Decimal Value.* This is the latitude of the resource, in degrees.
169 
170 - `fuzz_factor=`
171 
172  *Floating-Point Decimal Value.* This is the value, in kilometers, of the location obfuscation. Setting this to zero (empty) will turn location obfuscation off.
173 
174 - `can_see_through_the_fuzz=`
175 
176  *Integer.* This is a security token that is allowed to see the "true" location of a location-obfuscated place.
177 
178 - `read_token=`
179 
180  *Integer.* This is a security token that is allowed to see this resource. You are allowed to set this to 0 (everyone can see) or 1 (all logged-in users can see, but only logged-in users).
181 
182 - `write_token=`
183 
184  *Integer.* This is a security token that is allowed to modify this resource. You can set this to 1 (all logged-in users can modify), but not 0.
185 
186 - `address_venue=`
187 
188  *String.* This is an arbitrary string value that names the venue of any address.
189 
190 - `address_street_address=`
191 
192  *String.* This is an arbitrary string value that names the street address of any address.
193 
194 - `address_extra_information=`
195 
196  *String.* This is an arbitrary string value that is for the "extra information" section of any address.
197 
198 - `address_town=`
199 
200  *String.* This is an arbitrary string value that names the town/city/municipality of any address.
201 
202 - `address_county=`
203 
204  *String.* This is an arbitrary string value that names the county or sub-province of any address.
205 
206 - `address_state=`
207 
208  *String.* This is an arbitrary string value that names the state or province of any address.
209 
210 - `address_postal_code=`
211 
212  *String.* This is an arbitrary string value that names the postal code of any address.
213 
214 - `address_nation=`
215 
216  *String.* This is an arbitrary string value that names the nation of any address.
217 
218 - `tag8=`
219 
220  *String.* This is an arbitrary string value that can be applied to the place.
221 
222 - `tag9=`
223 
224  *String.* This is an arbitrary string value that can be applied to the place.
225 
226 - `geocode`
227 
228  *No Value Required -Just Add the Query.* If this is specified, then the BAOBAB server will make its best effort to use the address information in the record to look up a longitude and latitude. It will save this long/lat (assuming it worked) in the record.
229 
230 - `reverse-geocode`
231 
232  *No Value Required -Just Add the Query.* If this is specified, then the BAOBAB server will make its best effort to use the long/lat information in the record to look up an address. It will save this address (assuming it worked) in the record.
233 
234 DELETE OPERATIONS
235 -----------------
236 Delete is quite simple. Just select one or more resources (either by direct selection, or by a search), and specify DELETE as the HTTP method.
237 
238 Any records within that set that are writeable by the current login will be deleted, and a detailed report will be returned in the chosen format.
